## 1. 대시보드 데이터 정확도 검증 결과

### 검증 방법
- ERP 마감통계관리 → 유종별매출이익분석 원본 CSV vs 대시보드 sales.csv 비교
- 2026년 Q1 (1~3월) 전수 대조

### 결과
| 구분 | ERP 원본 | 대시보드 CSV | 차이 |
|------|----------|-------------|------|
| 2026 Q1 매출액 | 5,695,161,495원 | 5,695,161,495원 | **0원** |

**결론: 대시보드 매출 데이터는 ERP 마감통계관리와 100% 일치**

### 경영속보와의 차이 (참고)
- 경영속보 BEX유통 Q1 누계: 약 72.8억
- 대시보드: 56.95억 (제품 49.2억 + 상품 7.7억)
- 차이 약 15.85억 → 경영속보는 별도 회계 기준 적용 (경영관리팀 확인 필요)

### 제품 vs 상품 구분
- **제품**: 유종대분류에 "(상)" 없음 → 자사 생산 제품
- **상품**: 유종대분류 "(상)" 접미사 → 상품 (외부 매입 후 재판매)
- 둘 다 대시보드에 포함되어 있음 (누락 없음)

## 2. 대시보드 현황 (v3.1)

### Phase 1 — 매출/이익 대시보드 (완료)
- 매출 추이 (월별/분기별)
- 팀별 매출 비교
- 거래처별 매출 순위
- 유종별 매출 분석
- 영업이익 분석
- 미수금 현황
- 목표 대비 실적 달성률

### Phase 2 — SCM 모듈 (진행 중)
5개 화면 신규 구축:
1. **상품 마스터** — 제품/상품/부자재/소모품 통합 관리
2. **거래처 관리** — 다중 담당자 지원
3. **BOM 관리** — MOQ별 단가 테이블
4. **소모품 관리** — 재고 추적
5. **발주 관리** — 발주서 작성/진행/완료 3단계

## 3. 데이터 파이프라인

```
ERP 마감통계관리
  → 유종별매출이익분석 CSV (EUC-KR 인코딩)
    → 정제 (부서코드 ZZZZZZ 합계행 제거, 콤마 제거)
      → public/data/sales.csv (UTF-8)
        → Next.js 빌드 시 정적 로드
```

### 데이터 파일
| 파일 | 설명 | 행 수 |
|------|------|-------|
| sales.csv | 매출/원가/이익 통합 | ~15,000+ |
| targets.csv | 월별 목표 매출/이익 | ~40 |
| receivables.csv | 미수금 현황 | ~50 |

### 팀 구분
- 영업지원팀
- 유통1팀
- 유통2팀
- 유통3팀

## 6. 주의사항

- ERP CSV는 **EUC-KR(CP949) 인코딩** → PowerShell에서 `[System.Text.Encoding]::GetEncoding(949)` 사용
- 큰따옴표 안의 콤마 숫자(`"15,000"`) 주의 → regex로 정제 필요
- 부서코드 `ZZZZZZ` / 거래처 `품목별합계` 행은 소계이므로 반드시 제외
- 경영속보와 ERP 마감통계는 기준이 다름 → 직접 비교 불가
