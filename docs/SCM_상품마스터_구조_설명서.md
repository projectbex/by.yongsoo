## 1. 개요

전사 통합 품번(바코드) 기준으로 Sister Ann 전 제품의 마스터 정보를 관리하는 페이지입니다.
상품 마스터는 이후 구축될 재고장, BOM, 발주 관리 등 SCM 모듈의 기반 데이터가 됩니다.

**경로**: `/dashboard/scm/products`

## 2. 데이터 구조

### 2-1. 상품 식별

| 필드 | 설명 | 비고 |
|------|------|------|
| **통합 품번 (uid)** | 전사 대표 키 | 고유값, 중복 불가 |
| 바코드 | EAN-13 바코드 | 중복/공란 허용 (동일 바코드 복수 SKU 가능) |

### 2-2. 상품 분류 (2단계)

| 필드 | 설명 | 예시 |
|------|------|------|
| **카테고리 (대분류)** | 부위/기능별 | EYE, LIP, FACE, SKINCARE, 기타 |
| **시리즈명** | 제품 시리즈 그룹 | 더블이펙트 워터프루프 아이펜슬, 멜로우 듀이 립 블러 등 |

- 시리즈명이 같은 SKU끼리 **서류/파일을 공유** (MSDS, 디자인 파일 등)
- 단독 제품은 시리즈명 공란

### 2-5. 원가 관리

| 필드 | 설명 |
|------|------|
| 현재 원가 (KRW) | 최신 이력에서 자동 반영 (수동 수정 불가) |

- 적용일 + 원가 + 메모를 등록하면 자동으로 현재 원가에 반영
- 향후 BOM 관리와 연동하여 원재료 기반 원가 산출 예정

## 3. 주요 기능

### 3-1. 목록 뷰
- 검색: 바코드, 상품명, 제조사, 시리즈명
- 필터: 상태, 카테고리, 시리즈명 (3중 필터 동시 적용)
- 정렬: 모든 컬럼 헤더 클릭 시 오름/내림차순 토글
- 기본 정렬: 등록 순서

### 3-2. 상세 패널 (3탭)
- **기본 정보**: 식별, 분류, 상품명, SKU, 원가, 원가 이력, 메모
- **스펙·체적**: 제품 스펙, 단상자/인박스/아웃박스, HS Code, 리드타임
- **서류·파일**: 시리즈 공통 파일 + 개별 파일 (2단계)

### 3-3. 수정 모드
- 읽기 ↔ 수정 모드 토글
- 이전/다음 상품 버튼 (목록 탐색)
- 통합품번/SKU 중복 자동 체크

### 3-4. 엑셀 연동
- **엑셀 업로드**: 통합품번 기준 부분 업데이트 (빈 셀 = 기존 유지)
  - 파일 선택 → 미리보기 (신규/수정 건수 + 변경 항목) → 확인 후 저장
- **엑셀 다운로드**: 현재 조회 결과 기준 전체 항목 xlsx 다운로드
- **템플릿 다운로드**: 업로드용 빈 양식

### 3-5. 서류 관리 (시리즈 공유)
- 시리즈명이 같은 제품들은 **시리즈 공통 파일** 공유
  - 예: 아이펜슬 15 SKU → MSDS 1번만 업로드하면 전체 공유
- 개별 제품 전용 파일도 별도 등록 가능
- 카테고리: MSDS, 인증서류, 디자인, 수출서류, 성분자료, 기타

## 4. DB 모델

```
ProductMaster (상품 마스터)
├── uid (통합품번, 대표 키, unique)
├── barcode (바코드, nullable)
├── status (운영상태)
├── category (대분류) + seriesName (시리즈명)
├── nameKo/En/Ja/Tw/Hk (5개 언어)
├── skuKr/Tw/Hk/Id (국가별 SKU)
├── costPrice (현재 원가 — 이력에서 자동 반영)
├── 스펙 (volume, weight, manufacturer, shelfLifeMonths, paoMonths)
├── 체적 (box/inbox/outbox 치수, 입수량, 중량)
├── hsCode, leadTimeDays, memo, photoUrl
└── 관계
    ├── ProductCostHistory[] (원가 이력)
    └── ProductDocument[] (서류/파일)

ProductCostHistory (원가 이력)
├── productId → ProductMaster
├── appliedDate (적용일)
├── costPrice (원가)
└── memo (메모)

ProductDocument (서류/파일)
├── productId → ProductMaster
├── category (series:msds = 시리즈공통, msds = 개별)
├── fileName, fileUrl, fileSize
└── uploadedAt
```
